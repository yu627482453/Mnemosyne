## 3.2 Schema 与元数据标准 ✅

### Frontmatter 完整定义

```yaml
---
id: "3001-00001"              # {目录编号}-{5位序号}
title: "Transformer"           # 知识标题
type: concept                  # concept | entity | topic | comparison
domain: "3001-深度学习"        # 所属主题目录名
tags: [NLP, 注意力机制, LLM]    # 8-10 个关键词
aliases: [Transformer模型]     # 别名
created: 2026-05-15            # 创建日期
updated: 2026-05-15            # 更新日期
source: manual                 # manual | url | file | claude
status: draft                  # draft | review | published
links: [3001-00002]            # 关联知识 ID
summary: "基于自注意力机制的序列建模架构"
---
```

### 字段详细定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | `{目录编号}-{5位序号}`，唯一标识 |
| `title` | string | ✅ | 知识标题，用作 Obsidian 页面名 |
| `type` | enum | ✅ | concept / entity / topic / comparison |
| `domain` | string | ✅ | 主题目录名，Base 分组主维度 |
| `tags` | list | | 8-10 个关键词 |
| `aliases` | list | | 别名，Obsidian 原生支持 |
| `created` | date | ✅ | YYYY-MM-DD |
| `updated` | date | ✅ | YYYY-MM-DD，每次修改时更新 |
| `source` | enum | | manual / url / file / claude |
| `status` | enum | ✅ | draft / review / published |
| `links` | list | | 关联知识 ID 列表 |
| `summary` | string | ✅ | 一句话摘要，≤50字 |

### type 定义

| 值 | 含义 | 内容结构特征 |
|----|------|-------------|
| `concept` | 概念/理论 | 定义 → 核心要点 → 详细说明 → 相关概念 |
| `entity` | 实体 | 信息表 → 关键事实 → 关联 |
| `topic` | 主题 | 概述 → 分层展开 → 子主题 → 参考来源 |
| `comparison` | 比较 | 对比概览表 → A 特点 → B 特点 → 适用场景 → 结论 |

### status 流转

```
draft ──→ review ──→ published
  │                      │
  └──────────────────────┘
         重新编辑
```

### 模板对应

| 模板 | type | 目标目录 |
|------|------|----------|
| `t-concept.md` | concept | 主题目录 |
| `t-entity.md` | entity | 主题目录 |
| `t-topic.md` | topic | 主题目录 |
| `t-comparison.md` | comparison | 主题目录 |
| `t-inbox.md` | — | `0003-inbox/` |
| `t-daily.md` | — | 日记目录 |

## 3.3 数据流转全流程 ✅

### 知识完整生命周期

```
                  ┌──────────────────────────────┐
                  │         0003-inbox/           │
                  │  手动 / URL / 文件 / 对话      │
                  └──────────────┬───────────────┘
                                 │
                    Claude 处理   │  Ingest
                                 ▼
                  ┌──────────────────────────────┐
                  │      主题目录                 │
                  │  {编号}-{序号}-{slug}.md      │
                  │  status: draft               │
                  └──────────────┬───────────────┘
                                 │
                    Claude/人工   │  Update
                                 ▼
                  ┌──────────────────────────────┐
                  │  status: review              │
                  │  人工审核通过                  │
                  └──────────────┬───────────────┘
                                 │
                                 │  Publish
                                 ▼
                  ┌──────────────────────────────┐
                  │  status: published           │
                  │  可被正式引用                  │
                  └──────────────────────────────┘

     Lint 定期检查 ←── 全流程覆盖
     Query 随时查询 ←── 全流程覆盖
```

### 五操作详述

| 操作 | 触发 | 输入 | 输出 | 副作用 |
|------|------|------|------|--------|
| **Ingest** | 用户提供 + Claude | inbox 文件/URL/文本 | 新知识条目 (draft) | 更新 INDEX.md、_counter.md、LOG |
| **Update** | 用户要求 | 已有知识 ID | 更新后条目 | 更新 updated 字段、LOG |
| **Publish** | 人工确认 | review 条目 | status→published | — |
| **Query** | 用户提问 | 自然语言 | 合成回答 + 引用 | 可选存档 outbox |
| **Lint** | 用户触发 | — | 问题报告 | 自动修复项直接执行 |

### Ingest 步骤

```
1. 素材放入 0003-inbox/
2. 用户 Claude Code: @"0003-inbox/xxx.md" 处理
3. Claude 读取 → 确定 Domain/Type → 读 _counter.md → 分配 ID
4. 按模板创建 {编号}-{序号}-{slug}.md，填充 Frontmatter + 正文
5. 更新 {目录}/INDEX.md（新增文件条目）
6. 更新 {目录}/_counter.md（递增）
7. 检查 _index.md 是否需要更新
8. 追加 0109-log/LOG-YYYY-MM-DD.md
9. 报告：新建 {id}，建议关联 {links}
```

### Query 步骤

```
1. 用户提问
2. Claude Grep/读 INDEX.md → 定位候选文件
3. 读候选文件 → 合成回答
4. 引用 [[wikilink]] 或 {id}
5. 用户可选："存档到 outbox"
6. 追加 LOG
```

### Lint 检查清单

**自动修复**：
- INDEX.md 与实际文件不一致 → 更新
- _counter.md 与最大 ID 不一致 → 修正
- 内部 [[wikilink]] 指向不存在 → 标记 %% 注释

**报告（需人工）**：
- 孤立页面（无 incoming link）
- 长期 draft >30 天
- 缺失跨主题引用建议
- Frontmatter 不完整
- summary 超长或缺失

### 级联更新规则（P1）

Update 时 Claude 检查：
1. 该知识的 `links` 列表 → 通知用户可检查相关页面
2. 其他页面 `links` 中引用该 ID → 建议更新
3. `links` 新增 ID → 在目标页面建议加回链

## 3.4 Git 版本控制策略 ✅

### 分支策略

日常直接在 master 提交（单人使用，trunk-based）。仅批量操作（影响 10+ 文件）时可选开分支验证后合并。

### 提交方式

全部手动提交，不使用自动提交插件。Claude 操作后由 Claude 直接执行 git 命令。

| 场景 | 方式 | 格式 |
|------|------|------|
| Claude 知识操作 | Claude 提交 | `wiki: {操作} {id} — {摘要}` |
| 项目文档变更 | 人工/Claude 提交 | `docs: {描述}` |
| 用户日常笔记 | 人工提交 | 自由描述 |

### .gitignore

```gitignore
.obsidian/
*.tmp
*.bak
.DS_Store
Thumbs.db
```

### Claude 操作后提交流程

```
完成操作 → git status 确认 → git add {文件} → git commit -m "wiki: ..." → git push
```

### 提交粒度

| 操作 | 变更文件 | 提交 |
|------|----------|------|
| Ingest | 知识文件 + INDEX.md + _counter.md + LOG | 一次提交 |
| Update | 知识文件 + LOG | 一次提交 |
| Lint | INDEX.md + 标记文件 | 一次提交 |
