# Phase 1 需求澄清 — 问答记录

> 日期：2026-05-12

## 第一轮

### Q1: MVP 阶段应该覆盖哪些核心能力？

**回答**：先实现最小 MVP，然后中等，最后实现完整 MVP。但在设计时需要提前考虑对应的扩展性。

### Q2: Wiki 初始阶段应该聚焦哪些知识领域？

**回答**：全部四个领域：
- 工作业务知识（4000-4999）
- AI 技术知识（3000-3999）
- 通用基础知识（5000-9999）
- 个人生活知识（1000-2999）

### Q3: 当前 Obsidian 的插件安装情况？

**回答**：已装部分插件（具体待确认）

### Q4: Git 远程仓库是否已就绪？

**回答**：先本地仓库，后配置远程 `git@github.com:yu627482453/Mnemosyne.git`

## 第二轮

### Q5: JD 唯一标识体系采用哪种格式？

**回答**：层级前缀+序号，但不需要 "JD-" 前缀。例如 `5000-00001`、`3000-00100`

### Q6: 知识摄入的主要来源有哪些？

**回答**：全部四种：
- 手动编辑
- URL 抓取
- 文件导入
- Claude 对话沉淀

### Q7: Claude Code 与 Obsidian 的交互方式偏好？

**回答**：手动触发处理（在 Claude Code 中指定文件/目录，由 Claude 处理）

### Q8: 项目推进中的确认粒度偏好？

**回答**：子步骤确认（每个阶段内的每个子步骤完成后都暂停等待确认）
