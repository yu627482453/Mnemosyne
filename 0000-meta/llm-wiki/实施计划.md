# 基于 Obsidian + Claude Code 的智能本地 Wiki 系统 — 实施计划

## Context

用户希望构建一个集知识自动采集、智能分类、深度关联、自然语言检索于一体的个人知识管理系统。以 Obsidian 为前端载体，Claude Code 为后端智能引擎，实现知识从摄入到应用的全流程自动化与智能化。

当前 vault（`D:\obsidian`）几乎是空的——仅含该项目的提示词文档和 `.obsidian/` 默认配置，无 git 提交记录。需要从零开始搭建整个系统。

---

## 一、核心设计思想（来自用户初步设计）

1. **渐进式知识披露**：通过 Obsidian Base 面板和结构化目录体系，实现知识的分层导航与按需访问
2. **LLM 知识管家**：Claude 作为整个知识系统的管理者，负责知识的加工、关联、维护和问答
3. **JD 唯一标识体系**：为每条知识分配唯一的数字 ID，实现精确检索、版本控制和关系管理
4. **三层数据架构**：从原始数据到结构化知识的渐进式加工流程
5. **Git 原生支持**：全系统版本控制，自动追踪知识变更历史

## 三、三层数据处理架构（来自用户初步设计）

### 1. 数据准备区（0003-inbox）
- 存储所有未处理的原始知识
- 支持手动输入、URL 抓取、文件导入等多种摄入方式
- 数据处理完成后可自动归档或删除
- 设计统一的原始数据格式规范

### 2. 数据基础层（5000-9999）
- 对原始数据进行清洗、标准化、去重处理
- 提取核心内容，去除无关信息
- 分配唯一 JD 标识和基础元数据
- 形成可复用的知识原子单元

### 3. 数据关系层（0100-0104）
- 对知识原子进行深度加工和语义理解
- 自动识别概念、实体、主题和比较关系
- 建立知识之间的双向链接和关联图谱
- 生成结构化的知识卡片和摘要

## 四、关键技术要求（来自用户初步设计）

### Git 版本控制
- 初始化 Git 仓库管理整个 Obsidian vault
- 设计合理的分支策略（主分支 + 功能分支）
- 实现知识变更的自动提交和推送
- 保留详细的提交日志，便于回溯

### Claude Code 集成
- 设计 Claude 与 Obsidian 的交互接口
- 实现知识自动处理的工作流脚本
- 支持自然语言查询和知识问答
- 确保所有 AI 操作可追溯、可审核

### Obsidian 插件配置
- 推荐并配置必要的核心插件
- 重点关注与 LLM 集成、数据库、图谱相关的插件
- 优化 Obsidian Base 面板配置，实现渐进式导航

## 五、项目交付流程

按照六个阶段逐步推进，**每个阶段完成后必须等待用户明确确认才能进入下一阶段**：

### 阶段 1: 明确目标与需求细化
- 基于当前需求，提出关键澄清问题
- 形成正式的需求规格说明书
- 确认功能优先级和 MVP 范围

### 阶段 2: 方案调研与技术选型
- 2.1 插件生态评估：审计现有 40+ 插件，核心/可选/冗余分类
- 2.2 Obsidian 原生能力调研：Base（数据库视图）、Canvas（白板）、CLI（命令行接口）
- 2.3 karpathy-llm-wiki 调研：编译模式 vs RAG，与三层架构对比
- 2.4 Claude Code × Obsidian 集成方案：CLI 命令 + 文件读写对比
- 2.5 知识分类与关联方法：目录 vs 标签 vs Frontmatter vs Base，wikilink vs MOC vs Dataview vs Canvas
- 2.6 知识模板详细设计：6 种类型 Frontmatter 和 Templater 变量
- 形成技术选型报告

### 阶段 3: 详细方案设计
- 完善目录结构和文件命名规范
- 设计知识 Schema 和元数据标准
- 定义数据流转和处理流程
- 设计 Git 版本控制策略
- 绘制系统架构图和数据流图

### 阶段 4: 实施计划生成
- 分解任务为可执行的小步骤
- 估算每个任务的工作量
- 制定详细的项目时间表
- 识别潜在风险和应对措施

### 阶段 5: 方案实现与测试
- 搭建基础 Obsidian 仓库结构
- 配置必要的 Obsidian 插件
- 实现 Claude Code 集成脚本
- 开发知识自动处理工作流
- 进行功能测试和优化

### 阶段 6: 项目总结与文档
- 编写系统使用手册
- 总结项目经验教训
- 提出后续优化建议

## 六、Phase 1 待澄清的关键问题

### MVP 范围
1. MVP 阶段应该覆盖哪些核心能力？
   - 最小 MVP：目录结构搭建 + 手动笔记 + Git 版本控制
   - 中等 MVP：最小 MVP + 知识模板 + Claude Code 辅助内容加工
   - 完整 MVP：中等 MVP + 自动分类 + 知识关联图谱

2. Wiki 初始聚焦的知识领域？文档预留了 1000-9999 若干知识域，先启用哪些？

### 技术选型
3. 当前已安装/偏好的 Obsidian 插件？（如 Dataview、Templater、QuickAdd、Dataview、DB Folder 等）
4. Git 远程仓库是否已就绪？需要 GitHub/GitLab 等托管吗？
5. JD 唯一标识体系的具体格式偏好？（时间戳、UUID、自增数字？）

### 内容与语言
6. Wiki 内容以中文为主？是否有英文内容需求？
7. 知识摄入的主要来源？（手动编辑、URL 抓取、文件导入、其他？）

### 工作流偏好
8. Claude Code 交互方式：手动触发 Claude 处理笔记 vs Claude 自动监控 inbox 变化？
9. 进度确认粒度：每个子步骤确认 vs 每个大阶段确认？
