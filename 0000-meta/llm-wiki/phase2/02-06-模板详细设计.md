# 2.6 知识模板详细设计

> 日期：2026-05-14

## 一、模板总览

| 模板 | 文件名 | 用途 | 目标目录 |
|------|--------|------|----------|
| 概念模板 | `t-concept.md` | 术语定义、抽象概念、理论 | 主题目录 → 0101 关系层 |
| 实体模板 | `t-entity.md` | 人物、组织、项目、产品 | 主题目录 → 0102 关系层 |
| 主题模板 | `t-topic.md` | 跨概念综合论述、专题 | 主题目录 → 0103 关系层 |
| 比较模板 | `t-comparison.md` | A vs B 对比分析 | 主题目录 → 0104 关系层 |
| 收件箱模板 | `t-inbox.md` | 原始素材条目 | `0003-inbox/` |
| 日记模板 | `t-daily.md` | 日常记录 | 日记目录 |

## 三、Templater 变量定义

| 变量 | 来源 | 说明 |
|------|------|------|
| `{{title}}` | 用户输入 | 知识标题 |
| `{{id}}` | `_counter.md` 注册表 | 在当前主题目录的 `_counter.md` 中取最大序号 +1 |
| `{{domain}}` | 目录名 | 当前主题目录名，如 `3001-深度学习` |
| `{{tags}}` | Claude 建议 | 8-10 个关键词标签 |
| `{{aliases}}` | 用户输入 | 别名，分号分隔 |
| `{{source}}` | 用户选择 | manual\|url\|file\|claude |
| `{{url}}` | 用户输入 | 仅 source=url 时填充 |
| `{{links}}` | Claude 分析 | 关联的知识 ID 列表 |
| `{{summary}}` | Claude 生成 | 一句话摘要 |
| `{{compare_to}}` | 用户输入 | 比较模板专用，对比对象 |

## 四、模板存放与加载

```
0000-meta/0001-templates/
├── t-concept.md
├── t-entity.md
├── t-topic.md
├── t-comparison.md
├── t-inbox.md
└── t-daily.md
```

**加载方式**：
- **Templater 插件**：设置模板目录为 `0000-meta/0001-templates`
- **QuickAdd**：为每个模板创建 Capture 命令，一键选择模板创建笔记
- **Claude Code**：通过 CLI `obsidian create name="..." template="t-concept"` 使用模板

## 五、2.6 结论

6 个模板覆盖了知识从摄入到存储的完整流程：

```
0003-inbox/                      t-inbox.md
     │                              原始素材
     ▼
{编号}-{主题名}/{id}-{slug}.md  t-concept / t-entity / t-topic / t-comparison
     │                              结构化知识
     ▼
0101-0104/ + 0105-wiki-base/    Base 视图 + MOC + Canvas
                                    关系层导航
```

**关键决策**：
- 所有知识类型共用同一套 Frontmatter Schema（id/type/domain/tags/links/summary）
- 内容结构按类型分化（概念=定义式 / 实体=信息表 / 主题=分层展开 / 比较=双栏对比）
- 模板既可通过 Obsidian 手动使用，也可通过 Claude Code CLI 程序化调用
