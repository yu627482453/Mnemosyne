# 2.6 知识模板详细设计

> 状态：模板细节文档。当前总方案以 `../CURRENT.md` 为准。

> 日期：2026-05-14 | 修订：2026-05-19（模板按 L2/L3 分层收束）

## 一、模板总览

模板分两组：L2 负责沉淀来源约束的标准化主题知识；L3 负责主题、概念、实体、对比等加工页面。

| 层级 | 模板 | 文件名 | 适用特征 | 目标目录 |
|------|------|--------|----------|----------|
| L2 | 标准化知识模板 | `t-knowledge.md` | L1 原始素材标准化后的主题知识条目 | 主题目录 |
| L1 | 收件箱模板 | `t-inbox.md` | 原始素材条目 | `0003-inbox/` |
| 日常 | 日记模板 | `t-daily.md` | 日常记录 | 日记目录 |
| L3 | 主题综述模板 | `s-topic.md` | 跨 L2 标准化主题知识的专题综述 | `0101-wiki-topics/` |
| L3 | 概念加工模板 | `s-concept.md` | 某主题域的概念解释/概念索引 | `0102-wiki-concepts/` |
| L3 | 实体档案模板 | `s-entity.md` | 人物/组织/项目/产品档案 | `0103-wiki-entities/` |
| L3 | 对比分析模板 | `s-comparison.md` | 两个或多个对象的系统对比 | `0104-wiki-comparisons/` |

> 关键收束：`concept/entity/topic/comparison` 不再作为 L2 主题目录的模板类型，而是 L3 知识加工层的页面形态。L2 主题目录统一使用 `t-knowledge.md`。

## 五、变量定义

| 变量 | 来源 | 说明 |
|------|------|------|
| `{{title}}` | 用户/Claude | 页面标题 |
| `{{slug}}` | 标题生成 | 文件名与 wikilink 使用的可读 slug |
| `{{topic}}` | 目录名 | L2 所属主题目录 |
| `{{layer}}` | 模板默认 | L2 / L3 |
| `{{kind}}` | 模板默认 | L2 为 standard；L3 为 topic/concept/entity/comparison |
| `{{tags}}` | Claude 建议 | 3-10 个关键词标签（推荐 8+），优先使用受控词表 |
| `{{aliases}}` | Claude/用户 | 包含 slug 的别名列表 |
| `{{source}}` | 用户选择 | manual / url / file / claude |
| `{{summary}}` | Claude 生成 | 一句话摘要 |
| `{{processing_path}}` | Claude 生成 | L3 加工页面路径维度：`{大类}/{主题域}` |
| `{{entry_count}}` | Claude 统计 | L3 加工页引用的 L2 条目数量 |
| `{{source}}` | Claude 统计 | L3 加工页引用的 L2 源文件路径列表 |

## 六、模板存放与加载

```
0000-meta/0001-templates/
├── t-knowledge.md
├── t-inbox.md
├── t-daily.md
├── s-topic.md
├── s-concept.md
├── s-entity.md
└── s-comparison.md
```

**加载方式**：
- **Templater 插件**：设置模板目录为 `0000-meta/0001-templates`
- **QuickAdd**：P0 只配置 `t-knowledge.md` 和 `t-inbox.md`
- **Claude Code**：L2 Ingest 使用 `t-knowledge.md`；L3 Process 使用 `s-*` 模板

## 七、2.6 结论

```
0003-inbox/                         t-inbox.md
     │                                 原始素材
     ▼
{编号}-{主题名}/{slug}.md          t-knowledge.md
     │                                 L2 标准化主题知识
     ▼
0101-0104/                         s-topic / s-concept / s-entity / s-comparison
                                       L3 知识加工层
```

**关键决策**：
- L2 主题目录统一使用 `t-knowledge.md`，只存放从 L1 标准化后的主题知识。
- L3 知识加工层使用独立的 `s-*` 模板，承载 topic/concept/entity/comparison 四类页面。
- `kind` 保留，但含义随层级变化：L2 固定为 `standard`，L3 表示加工页面形态。
