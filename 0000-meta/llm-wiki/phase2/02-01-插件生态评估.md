# 2.1 插件生态评估

> 日期：2026-05-12 | 当前插件总数：41

---

## 一、核心插件（Wiki 系统直接依赖 — 8 个）

| 插件 | 用途 | Wiki 系统中的角色 |
|------|------|--------------------|
| **templater-obsidian** | 高级模板引擎 | 6 种知识模板的引擎，支持动态变量和脚本 |
| **dataview** | 动态索引查询 | INDEX.md 的查询引擎，自动聚合知识列表 |
| **dbfolder** | 结构化数据管理 | 辅助数据基础层的表格化管理 |
| **obsidian-git** | Git 版本控制 | 知识变更追踪、自动提交、历史回溯 |
| **omnisearch** | 全文检索 | 跨目录快速定位知识条目 |
| **metaedit** | 元数据编辑 | 批量修改 frontmatter（id、tags、status 等） |
| **live-variables** | 实时变量 | 与 Templater 配合，模板中直接显示动态计算值 |
| **obsidian-custom-attachment-location** | 附件路径管理 | 确保粘贴的图片/文件自动归入 `0001-resource/` |

## 四、建议禁用（功能重叠或无关 — 22 个）

功能重叠类：
- drawio-obsidian（与 excalidraw 重叠）
- highlightr-plugin（与 dynamic-highlights 重叠）
- markdown-table-editor（与 table-editor-obsidian 重叠）
- obsidian-chartsview-plugin（与 echarts 重叠）
- obsidian-vault-changelog（Git 已覆盖）
- card-board（与 kanban 重叠）
- emoji-shortcodes / obsidian-emoji-toolbar（二选一）

与 Wiki 系统无关或极少使用：
- game-search、ledger-obsidian、webpage-html-export
- obsidian-markdown-export-plugin、obsidian-pandoc
- obsidian-plantuml、text-extractor
- obsidian-contextual-typography、obsidian-style-settings
- obsidian-collapse-all-plugin、obsidian-bartender
- file-tree-alternative、workspaces-plus、obsidian42-brat
- mousewheel-image-zoom、markdown-prettifier

## 五、建议新增安装

| 插件 | 用途 | 优先级 |
|------|------|--------|
| **QuickAdd** | 一键选择模板创建笔记、运行宏 | 🔴 高 |
| **Tag Wrangler** | 标签批量重命名/合并 | 🟡 中 |
| **Note Refactor** | 将长笔记按标题拆分为子笔记 | 🟡 中 |
| **Commander** | 自定义命令按钮到工具栏 | 🟢 低 |

## 六、结论

| 分类 | 数量 | 建议 |
|------|------|------|
| 核心 | 8 | 必须保留 |
| 重要辅助 | 15 | 建议保留 |
| 可选保留 | 10 | 按需开启 |
| 建议禁用 | 22 | 清理减负 |
| 建议新增 | 4 | QuickAdd 优先 |

**核心判断**：当前插件体系相当完整，主要问题是冗余插件太多（22 个可关），而非能力不足。建议优先清理冗余，再安装 QuickAdd 补强快速捕获能力。
