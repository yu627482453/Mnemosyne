# 2.5 知识分类与关联方法

> 日期：2026-05-14 | 修订：2026-05-18（删除type、domain→topic、0101-0104改为合成层）

## 一、分类体系设计

### 单轴分类模型

知识按**主题目录**分类，不再预标注 type。类型（概念/实体/主题/对比）由 Wiki 合成层的内容形态决定，而非存入时预判。

- **Domain（领域）** = 主题目录编号的千位：千位 1-2 个人生活 / 3 AI技术 / 4 工作业务 / 5+ 通用基础
- 知识原子无 type 标记，按内容结构特征选择模板

### 各分类方式的角色

| 方式 | 角色 | 实现 |
|------|------|------|
| **主题目录** | 物理存放位置 | `{编号}-{主题名}/`，如 `3001-深度学习/` |
| **Frontmatter topic** | 所属主题目录名，Base 分组维度 | `topic: "3001-深度学习"` |
| **ID 前缀** | 编号即目录号，千位隐含领域 | `3001-00001` → AI技术 |
| **Frontmatter tags** | 灵活多维标签（P3 重点建设） | `tags: [tag1, tag2]` |
| **Base 视图** | GUI 导航，按 topic/tag 过滤 | `0105-wiki-base/*.base` |

### 主分类方案：主题目录

**知识文件存放规则**：
- 文件存放在对应主题目录下，目录内平铺
- 文件命名：`{编号}-{5位序号}-{slug}.md`

```
3001-深度学习/
├── 3001-00001-transformer.md
├── 3001-00002-cnn.md
└── ...
```

### 0101-0104 合成层的定位

0101-0104 是 Claude 主动写作的合成知识层。内容从主题目录的知识原子提取、重组、扩写，形成可独立阅读的知识成品。

| 目录 | 内容 | 输出形态 |
|------|------|----------|
| `0101-wiki-topics` | 主题综述 | 跨概念综合论述 |
| `0102-wiki-concepts` | 概念解释 | 某主题域关键概念的体系化说明 |
| `0103-wiki-entities` | 实体档案 | 人物/组织/项目的完整档案 |
| `0104-wiki-comparisons` | 对比分析 | 两个或多个对象的系统对比 |

### 三层数据流中的分类位置

```
0003-inbox/                    原始素材，未分类
       ↓
{编号}-{主题名}/               数据基础层，知识原子
       ↓
0101-0104/                     Wiki 合成层，Claude 主动写作
```

## 二、关联策略设计

### 关联方式对比

| 方式 | 性质 | 优势 | 劣势 |
|------|------|------|------|
| **[[wikilink]]** | 持久化双向链接 | Obsidian 原生，Graph 可视化，反向链接自动追踪 | 需手动维护 |
| **Dataview 查询** | 动态自动聚合 | 可选附加于 INDEX.md 或合成页 | 学习成本 |
| **Canvas 连线** | 视觉探索 | 空间化思维 | 连线不生成 wikilink |
| **Base 视图** | GUI 过滤/分组 | 零代码 | 不如 Dataview 灵活 |

### 组合方案

```
持久化关联：[[wikilink]]
    知识页面间互相引用，单一数据源
    ↓
结构化导航：合成层 + Base
    0101-0104 合成页面做主题入口，Base 做按 topic/tag 过滤
    ↓
动态发现：Grep
    Grep 直搜 title/summary，定位候选文件
    ↓
视觉探索：Canvas
    专题梳理、头脑风暴、关系总览
```

### 关联的层级

| 层级 | 方式 | 示例 |
|------|------|------|
| **页面内** | `[[wikilink]]` | `transformer 中使用了 [[自注意力]] 机制` |
| **合成页** | 0101-0104 Claude 写作 | `- [[3001-00001]] — 基于自注意力...` |
| **跨主题索引** | Grep 直搜 | 直接搜索 title/summary 字段 |
| **入口导航** | Base in 0105 | 按 topic/标签过滤浏览 |
| **关系图谱** | Graph + Canvas | 可视化 wikilink 网络 |

## 三、具体实现规则

### 3.1 知识创建时的分类决策

Claude 处理 inbox 时按以下规则决定分类：

1. 确定 **Domain**：内容属于个人生活/AI技术/工作业务/通用基础？
2. 确定 **归属主题目录**：已有目录中匹配，匹配不到则新建
3. 确定 **ID**：Glob 目标目录取最大序号 +1
4. 确定 **Tags**：提取 8-10 个关键词标签
5. 识别 **wikilink**：从内容中建立到已有知识页面的 [[wikilink]]

### 3.2 wikilink 使用规范

```markdown
# 推荐：使用页面标题
transformer 中的 [[自注意力机制]] 是核心创新

# 不推荐：使用路径
transformer 中的 [[3001-00002-self-attention|自注意力]]
```
