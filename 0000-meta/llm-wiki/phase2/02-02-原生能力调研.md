# 2.2 Obsidian 原生能力调研

> 日期：2026-05-12 | 调研对象：Base / Canvas / CLI / Graph / Properties

## 一、Base（数据库视图）

- **版本**：v1.9.0 起核心插件（2025.05），当前 Obsidian v1.12.7
- **当前**：✅ 已启用
- **文件格式**：`.base`（YAML），不修改 `.md` 源文件
- **定位**：GUI 驱动的结构化数据管理，类 Notion 数据库

### 已发布能力

| 能力 | 说明 |
|------|------|
| Table / Card / List / Map View | 四种视图，Table 为基础 |
| GUI 过滤/排序/分组 | 无需代码，菜单操作 |
| 公式列 | `contains()`、`date()`、`file.hasTag()` 等 |
| Summaries | 总数/平均/计数 |
| Base Search | 在 Base 内搜索 |
| Bases API | 社区插件扩展新视图 |

### 与 Dataview 互补定位

| 维度 | Base | Dataview |
|------|------|----------|
| 操作 | GUI | DQL 查询语言 |
| 学习成本 | 低 | 中高 |
| 灵活性 | 中 | 高 |
| 维护 | Obsidian 官方 | 社区（减缓） |

**Wiki 中角色**：`0105-wiki-base/` 面板入口，按 topic/tag 建 `.base` 浏览视图。Base 做日常浏览，Dataview 做主题 `_index.md` 中的目录内查询。

## 二、Canvas（白板）

- **版本**：v1.1 起核心插件（2022.12）
- **当前**：✅ 已启用
- **文件格式**：`.canvas`（JSON Canvas 开放规范，MIT 许可）

### 核心能力

| 元素 | 说明 |
|------|------|
| 文本卡片 | 双击创建，内容存于 `.canvas` |
| 笔记卡片 | 拖入 `.md`，双向联动编辑 |
| 媒体卡片 | 图片/PDF/网页嵌入 |
| 连线 | 拖拽创建带标签箭头线 |
| 分组 | 框选多卡，命名+颜色 |
| 转换 | 文本卡片→正式笔记 |

**关键限制**：连线是**纯视觉标注**，不自动创建 `[[wikilink]]`。适合探索性梳理，wikilink 才是持久化关联。

**Wiki 中角色**：`0100-wiki-meta/RELATIONS.canvas` 知识关系总览，各知识域可建专题 canvas。

## 三、CLI（命令行）

- **版本**：v1.12.4 起内置（2026.02），需 Catalyst 许可证
- **当前**：✅ 用户已在设置中启用（需重启终端生效）
- **约束**：Obsidian 桌面端必须运行

### 关键命令

```bash
obsidian search query="关键词"
obsidian create name="标题" template="模板名"
obsidian read file="笔记名"
obsidian append file="笔记名" content="追加"
obsidian backlinks path="笔记.md" counts
obsidian links path="笔记.md"
obsidian daily:append content="- [ ] 待办"
obsidian tags counts
```

### 集成评估

| 维度 | CLI | 文件直读写 |
|------|-----|-----------|
| 全文搜索 | ✅ | ❌ 需 grep |
| 链接分析 | ✅ backlinks | ❌ 需解析 |
| 模板 | ✅ create --template | ❌ |
| 依赖 | ❌ Obsidian 必须运行 | ✅ 无 |
| 许可证 | ❌ Catalyst | ✅ |

**推荐**：文件直读写为主通道，CLI 补充链接分析和模板创建。

## 四、Graph（图谱）& Properties（属性）

两者均已启用，无需额外配置：

- **Graph**：双向链接可视化，自动展示 `[[wikilink]]` 关系网
- **Properties**：Frontmatter 的 GUI 编辑器，直接可视化管理 id/tags/type/status 等字段

## 五、2.2 结论

| 能力 | Wiki 角色 | 优先级 |
|------|----------|--------|
| Base | `0105-wiki-base/` GUI 浏览入口 | P0 |
| Graph | wikilink 可视化（已有） | P0 |
| Properties | Frontmatter GUI 编辑（已有） | P0 |
| Canvas | 关系图谱可视化总览 | P1 |
| CLI | 辅助链接分析/模板创建 | P1 |

**关键决策**：
1. Base 做日常浏览，Dataview 做复杂索引查询，互补共存
2. Canvas 做关系探索入口，wikilink 做持久化关联
3. 文件直读写为 Claude Code 集成主通道，CLI 为辅助
