# skill-ingest：知识摄入（L1 → L2 → L3）

> 触发：用户 `@` 引用 inbox 文件要求处理

## 前置检查

1. 确认目标文件在 `0003-inbox/` 下且存在
2. 读取全文，提取标题、来源、核心内容

## 拆分判断（一个 inbox 含多个独立概念时）

满足以下条件则拆分为多个 L2 条目：
- 内容有独立标题层级（如 `## CNN`、`## ResNet`）
- 每个概念可独立成文（有定义 + 要点）
- 概念之间有明确边界

不拆分的情况：
- 内容紧密关联、互相依赖（如 A 是 B 的改进/变体）
- 各概念信息量不足独立成文（<3 个要点）
- 用户明确要求合并

## 执行步骤

### 1. 判断归属主题目录

- 从内容中识别领域（个人生活 / AI技术 / 工作业务 / 通用基础）
- 对照 `0000-meta/0003-configs/topics.yaml`
- 匹配已有主题目录：选择最相关的一个
- 无匹配：按 topics.yaml 规则创建新主题目录
  - 编号 = Glob 同千位已有目录取最大编号 +1
  - 格式：`{4位编号}-{中文主题名}`
  - 报告用户确认

### 2. 生成 slug（D009）

- 英文优先，从标题生成可读 slug
- 给出 3-5 个推荐选项
- 用 `rg --files "{主题目录}/"` 检查重名
- 重名时自动追加 `-2`、`-3`
- 用户可确认推荐或手动指定

### 3. 创建 L2 标准化主题知识

按 `0000-meta/0001-templates/t-knowledge.md` 模板写入，填写规则：
- `tags`：3-10 个，优先用 `tag-vocabulary.yaml` 已有标签；新增标签需用户确认
- `aliases`：至少包含 slug、中文译名、常见变体
- `summary`：≤80 字
- `created` / `updated`：使用当天日期
- 正文至少包含：核心内容、要点（3 条以上）、关联（[[wikilink]]）、来源

### 4. 判断 L3 触发（D004/D013）

L3 页面结构：`010{1-4}/{大类}/{主题域}/{slug}.md`（0101 为域级 `{大类}/{主题域}.md`）

优先判断是否同时满足多种页面形态：
- **entity**：具体产品 / 项目 / 组织 / 人物 / 工具名词，且正文主要在说明“它是什么、怎么工作、边界与属性”
- **comparison**：正文主轴是两个或多个对象的差异、取舍、适用场景
- **concept**：正文主轴是机制、方法、原则、模式、抽象概念

同一条 L2 可同时触发多种 L3 页面；不要默认只落 concept。

| 情况 | 动作 |
|------|------|
| 已有同名概念页（同 slug） | 合并更新 `0102`（见 §合并规则） |
| 已有同名实体页（同 slug） | 合并更新 `0103` |
| 已有同名对比页（同 slug） | 合并更新 `0104` |
| 命中 concept 且无同名页 | 新建 `0102-wiki-concepts/{大类}/{主题域}/{slug}.md` |
| 命中 entity 且无同名页 | 新建 `0103-wiki-entities/{大类}/{主题域}/{slug}.md` |
| 命中 comparison 且无同名页 | 新建 `0104-wiki-comparisons/{大类}/{主题域}/{slug}.md` |
| 主题域首次有内容 | 新建 `0101-wiki-topics/{大类}/{主题域}.md`（主题综述）。0103/0104 目录按需创建，不预建空壳 |

L3 页面格式见 `CLAUDE.md` L3 Frontmatter 节和 `s-*.md` 模板。

0101 主题综述更新：域已有 0101 时，在"概念目录"区追加 `- [[{slug}]] — {summary}`，不要求重写概述。

### 5. L3 合并规则（级联更新）

当新 L2 与已有 L3 概念页匹配（同 slug）时：

1. 读入已有 L3 页面全文
2. 对比新 L2 内容：
   - 新事实/信息 → 补充到对应段落，保留已有内容
   - 冲突信息 → 在"争议与分歧"区块标注（见 §冲突检测）
   - 重复信息 → 不修改
3. 在 `source` 字段追加新 L2 路径
4. 在"关键来源"区追加新 L2 条目引用
5. 更新 `updated` 日期

### 6. 冲突检测

当多个 L2 来源对同一事实有不同表述时，在 L3 页面"争议与分歧"区块标注：

- 数值/日期冲突 → 标注两个来源的具体数值
- 定性描述矛盾 → 标注"观点分歧"
- 不确定时保守处理 → 保留已有内容，添加注释

### 7. 跨主题引用建议

1. 提取新 L2 的 tags
2. 搜索其他主题目录中 tags 重叠（≥2 个）的 L2 条目
3. 重叠且无 wikilink 连接 → 在 L2 正文中添加 `[[wikilink]]`
4. 在报告中列出建议，用户确认后执行

### 8. 追加操作日志

```markdown
| HH:MM:SS | Ingest | {主题目录}/{slug}.md | 新增 {标题} |
```

写入 `0109-log/LOG-YYYY-MM-DD.md`（不存在则创建）。

### 9. 报告与收尾

报告内容：新建文件路径、归属主题目录、L3 创建/更新情况、冲突标注（如有）、跨主题关联建议。
询问："是否移入回收站？"（D008：默认移入 `0003-inbox/.trash/`，执行前需用户确认）
用户确认后 → mv 到 .trash/ → 如用户明确要求，再执行 git add/commit/push
