# BM25：最佳匹配排序

## 定义

**BM25** 是基于词汇匹配的排名函数，通过精确单词/短语匹配检索文档，基于 [[tf-idf]] 构建但引入长度归一化和词频饱和机制。

## 核心改进

```
BM25 = TF-IDF + 文档长度归一化 + 词频饱和函数
```

**解决问题**：
- 长文档词频天然高（不公平）
- 常见词权重过高
- 词频线性增长不合理

## 适用场景

✅ **擅长**：唯一标识符（错误代码 TS-999）、技术术语、专有名词  
❌ **局限**：无法理解同义词、语义关系

## 与 Embedding 对比

| 维度 | BM25 | [[embedding]] |
|------|------|---------------|
| 原理 | 词汇精确匹配 | 语义相似度 |
| 优势 | 精确、快速 | 理解同义、跨语言 |
| 劣势 | 无语义理解 | 可能错过精确匹配 |

## 混合检索

现代 [[rag]] 结合两者：

```
查询 → [BM25检索 + Embedding检索] → 排名融合 → Top-K
```

## Contextual BM25

[[contextual-retrieval]] 方法：为 chunks 添加上下文后再建立 BM25 索引，额外提升 14% 性能。
